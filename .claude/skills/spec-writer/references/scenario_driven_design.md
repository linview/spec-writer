# 用户场景推演（scenario_driven_design.md）

> 本文件描述 spec-writer 在 PRD 功能需求完成后引导用户推演场景的流程。

---

## 触发时机

PRD 功能范围（scope）完成之后、功能需求（features）之前。模板可通过 `（spec-writer 指令）` 标记触发此流程。

---

## 为什么重要

- 没有用户场景 → 方案设计缺少依据
- 验收标准定义模糊
- 开发过程容易偏离用户需求

---

```

│ PRD.scenario (6/16), next: PRD.features  │
└──────────────────────────────────────────┘

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
用户场景推演
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

基于功能范围（In Scope）：
  - 功能 1：____
  - 功能 2：____
  - 功能 3：____

引导推演 3-5 个典型用户场景。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
场景 1/3：____
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【用户画像】
  - 角色：____（开发人员 / 测试人员 / SRE / ...）
  - 技能水平：____
  - 典型工作流：____

【场景描述】
  用户在什么情况下，遇到什么问题，需要使用这个功能。

【推演问题】
  Q1: 用户如何触发？____
  Q2: 需要提供哪些参数？____
  Q3: 系统如何响应？____
  Q4: 用户如何知道结果？____

[记录场景详情...]
```

## 场景推演模板

```markdown
### 场景 {编号}：{场景名称}

**用户画像**：
- 角色：{开发人员/测试人员/SRE/...}
- 技能水平：{熟悉 / 初学者 / ...}
- 典型工作流：{简述}

**场景描述**：
{用户在什么情况下遇到什么问题}

**操作步骤**：
1. 步骤 1：{用户操作}
   - 输入：{参数}
   - 系统响应：{反馈}

2. 步骤 2：...

**预期结果**：
- 成功：{期望输出}
- 失败：{错误提示、降级方案}

**验收标准**（从场景提取）：
- [ ] 验收条件 1
- [ ] 验收条件 2
```

## 推演维度检查清单

| 维度 | 推演问题 | 目的 |
|------|---------|------|
| 触发条件 | 用户在什么情况下使用？ | 明确使用场景边界 |
| 操作流程 | 用户需要执行哪些步骤？ | 设计交互流程 |
| 输入参数 | 用户需要提供什么？ | 定义接口参数 |
| 系统响应 | 系统如何反馈？ | 定义响应格式 |
| 成功情况 | 期望的输出？ | 定义成功标准 |
| 失败情况 | 可能的错误？ | 定义错误处理 |
| 边界条件 | 资源不足/超时/并发时？ | 定义降级方案 |

---

## 质量检查

- [ ] 覆盖了所有主要用户角色？
- [ ] 场景符合用户日常工作流程？
- [ ] 每个场景可以转换为验收标准？
- [ ] 考虑了异常情况？
- [ ] 验收标准明确可测试？
